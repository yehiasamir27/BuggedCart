Logging Security Issues

Problem 1: Logging Sensitive Data

Never log passwords, credit cards, PII, or session tokens in plaintext.

# BAD - Logs password
logger.info(f"User login: {username}, password: {password}")

# GOOD - Redacts sensitive data
logger.info(f"User login: {username}, password: [REDACTED]")

Problem 2: Insufficient Logging

Not logging security events makes incident response impossible.

  • Failed login attempts
  • Access control failures
  • Input validation failures
  • Server errors with user context

Problem 3: No Monitoring/Alerting

Logs are useless without active monitoring and alerting systems.

Prevention:

  • Implement centralized logging (ELK, Splunk)
  • Set up alerts for suspicious patterns
  • Redact sensitive data before logging
  • Log security events comprehensively
  • Regular log review and analysis
